Different URLs Provide Different Results

The YEG Neighbourhood Open Data Server ("script") can return different results, depending on values that you add to the end of the URL (address).

There are four main types of results this server will return:

  1. help (human-readable html)
  2. listDatasetNames (html or json)
  3. listNeighbourhoodNames (html or json), and
  4. getData (html or json).
Notes: .

How URLs are Built

The URL is the entire address you see in your browser's location bar.

For this version of this script, the [URL] is: .

We can add query parameter name/value pairs to the end of [URL] after a ?; if we use query parameters and values that the script understands, this script will return different results.

Query params are added in name=value pairs; a question mark comes before the first pair and ampersands separate the pairs after that: [URL]?queryParamName1=queryParamValue1&queryParamName2=queryParamValue2&...

Notes: on this page we often use [URL] instead of providing the entire long URL you see above.

How URLs are Built

The URL is the address you see in your browser's location bar.

By changing the query parameters, you will get different results back.

URLs are built as follows: [URL]?queryparamname1=queryparamValue1&queryParamName2=queryParamValue2&...

For further explanation, see.

What you are looking at right now is the default or help mode. This help html is displayed when no query parameters are added after [URL].

Visiting [URL]?mode=listDatasetNames provides valid values for datasetName and functionName. All datasets support functionName=getDataForNeighbourhoodName; some also support functionName=getChartForNeighbourhoodName.

Visiting [URL]?mode=listNeighbourhoodNames provides valid values for neighbourhoodName query parameter.

Visiting [URL]?mode=html provides a bootstrap v4.1 card suitable for embedding in a google site and requires the additional datasetName and functionName parameter values that you get from the two middle list views.

Visiting [URL]?mode=json provides a json object suitable for a google apps script app or MIT App Inventor app and requires the additional datasetName and functionName parameter values that you get from the two middle list views.
